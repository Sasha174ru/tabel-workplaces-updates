## Правило правок

1. Править сначала файлы в папке:
   `D:\Табель учёта рабочих мест\Программа`

2. Главные файлы:
   - `app.js` - логика приложения.
   - `styles.css` - внешний вид.
   - `index.html` - структура страниц.
   - `supabase-config.js` - настройки Supabase.

3. После каждой правки обязательно увеличить версию в `app.js`:
   `const APP_VERSION = "2026.07.13.xx";`

4. Эту же версию обязательно установить в Android-проекте `app\build.gradle`:
   - `versionCode` - версия без точек, например `2026071307`;
   - `versionName` - полная версия, например `2026.07.13.07`.

5. После правки скопировать эти файлы в Android assets:
   `D:\Табель учёта рабочих мест\Андроид\APK проект\app\src\main\assets\tabel`

6. Не трогать рабочие данные пользователей без отдельной необходимости.

## Публикация обновления

В GitHub-репозитории обновлений держать только один актуальный стабильный релиз:

- APK в релизе, например `default-2026071307-centered-columns.apk`;
- `default.apk`, который должен совпадать с текущим APK;
- `latest.json`;
- `README.md`;
- эту инструкцию, если она опубликована.

Старые релизы, теги и APK не хранить, чтобы телефон не путался и репозиторий не разрастался.

`latest.json` должен указывать на новый APK и новую версию.

## Что проверять руками после сборки

1. Приложение открывается на телефоне.
2. На старте проходит синхронизация.
3. Кнопка `Сохранить` показывает, что данные сохранены и синхронизированы.
4. Приём сотрудника:
   - сотрудник появляется в едином штате;
   - появляется в телефонах;
   - доступен для добавления на рабочее место.
5. Увольнение сотрудника:
   - исчезает из активного штата;
   - не мешает принять его снова;
   - старые отработанные смены остаются в статистике и бланках.
6. Рабочие места:
   - видно всех добавленных;
   - строку можно сдвинуть влево для удаления;
   - правила по разрядам работают.
7. Единый штат:
   - статус берётся по выбранной смене графика;
   - если сотрудник внесён в смену, статус `Работает`;
   - если не внесён, статус `Отсутствует`;
   - если выбрана причина отсутствия, показывается причина.
8. Бланки:
   - на Android нет кнопки печати;
   - на ПК печать открывается нормально;
   - строки и колонки не разъезжаются.
